This is ocamljs, a Javascript back-end for ocamlc, version 0.1.

What works (partial list)

  • The core of OCaml including the module system
  • Much of the standard library (including Printf)
  • Tail recursion
  • Generated code runs in Firefox/SpiderMonkey

We are using ocamljs for a production Firefox extension at Skydeck, but implementation has been on an as-needed basis.

What doesn't work or is unknown to work (partial list)

  • The object system
  • Floats need some work
  • int64
  • Much of the standard library
  • Untested in other Javascript implementations

Contributing

If you find a bug, it would be very helpful to get a test case that produces different behavior from regular OCaml (see test/ for examples). Other contributions are very welcome as well. For now I'd like to shoot for getting as much of OCaml and the standard library working as possible, then focus on performance.

Future plans

  • Compactify generated code
  • Dead code elimination
  • Improve code generation, inlining
  • CPS/closure-convert to first-order rep? Threads?
  • Something like Google Web Toolkit
  • Port the toplevel to ocamljs for a 'try OCaml' page
